---
title: 404 / Not Found page
---

The `404` or `Not Found` page is the page shown when a requested page does not exist.

## Steps on how to create your own.

### Create your page

Create your own `404` [page](page.md).

In server rendering, this page receives the Web Api request as props

Example: Create the page`./pages/404.tsx` with the following content:

```tsx
export const frontmatter = {
    "layout": "hamburger",
    "title": "Not Found"
}

export default function NotFound({request}: { request: Request }) {

    return <p>Error: The page {new URL(request.url).pathname} was not found</p>

}
```

### Register

The `404` page is available as the `NotFound` [page component](component.md#type).
You need to register it in the [configuration file](conf.md)

Example:
```json
{
  "components": {
    "NotFound": {
      "importPath": "./pages/404.js",
      "type": "page"
    }
  }
}
```

## Static Rendering

If a `404` page was not defined, we generate still one at `/404`
to conform to the hosting platforms, such
as [GitHub Pages](https://docs.github.com/en/pages/getting-started-with-github-pages/creating-a-custom-404-page-for-your-github-pages-site)
